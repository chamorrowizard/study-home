MEMORY MANAGEMENT

Understanding Ownership

Ownership is Rust’s most unique feature. It enables memory safety without a garbage collector.

4.1 What is Ownership?

In other languages, memory is managed via a Garbage Collector (Java/JS) or manually (C/C++). Rust uses a third approach: Ownership.

The Three Golden Rules

  1. Each value in Rust has an owner.
  2. There can only be one owner at a time.
  3. When the owner goes out of scope, the value will be dropped.

Stack vs. Heap

The Stack (Fast)
  • Stores data with fixed, known size.
  • LIFO (Last In, First Out) structure.
  • Pushing data is fast; no searching required.
The Heap (Flexible)
  • Stores data with dynamic size (e.g., String).
  • OS finds a hole and returns a pointer.
  • Slower access due to pointer following.

Variable Move vs. Clone

When you assign a heap-allocated variable to another, Rust performs a Move, not a shallow copy.

let s1 = String::from("hello");
let s2 = s1; // s1 is MOVED to s2. s1 is now INVALID.
// println!("{}", s1); // This would cause a compile error!
Deep Copy: If you actually want to copy the heap data, use let s2 = s1.clone();.

4.2 References and Borrowing

References (&) allow you to refer to a value without taking ownership of it.

Mutable vs. Immutable References

Type Syntax Capability
Immutable &T Read-only. Unlimited number allowed.
Mutable &mut T Read/Write. Only one allowed in scope.
The Big Restriction: You cannot have a mutable reference if you already have immutable references to the same value. This prevents Data Races. 
fn main() {
    let mut s = String::from("hello");
    change(&mut s);
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}

Dangling References

Rust prevents "Dangling Pointers" by ensuring the data outlives the reference. You cannot return a reference to a variable created inside a function.

4.3 The Slice Type

Slices let you reference a contiguous sequence of elements in a collection rather than the whole collection. They are a form of Reference.

let s = String::from("hello world");

let hello = &s[0..5]; // Range [start_index..end_index]
let world = &s[6..11];

String Slices as Parameters

Experienced Rustaceans use &str instead of &String in function signatures because it allows the function to accept both String references and string literals.

fn first_word(s: &str) -> &str { ... }
