Rust Chapter 3

A comprehensive guide to Variables, Data Types, Functions, Comments, and Control Flow.

3.1 Variables and Mutability

In Rust, variables are immutable by default. This encourages safety and concurrency.

Mutability

To make a variable mutable, you must use the mut keyword. This signals to future readers (and the compiler) that this value will change.

let mut x = 5;
println!("The value is: {x}");
x = 6; // This is only allowed because of 'mut'

Constants

Unlike immutable variables, constants (const) are always immutable. They must be annotated with a type and can be set only to a constant expression.

Shadowing

You can declare a new variable with the same name as a previous one. This is called shadowing. It allows you to change the type of a value while keeping the same name.

let spaces = "   ";
let spaces = spaces.len(); // Shadowing changes type from string to integer

3.2 Data Types

Rust is a statically typed language; it must know the types of all variables at compile time.

Scalar Types

  • Integers: Signed (i8-i128) and Unsigned (u8-u128).
  • Floating-Point: f32 and f64 (default).
  • Boolean: true and false.
  • Character: char (4-byte Unicode scalar value).

Compound Types

  • Tuples: Group multiple values of different types into one type. Fixed length.
  • Arrays: Group multiple values of the same type. Fixed length, stored on the stack.
let tup: (i32, f64, u8) = (500, 6.4, 1);
let (x, y, z) = tup; // Destructuring

3.3 Functions

Function signatures must declare the type of each parameter. This is a deliberate design choice in Rust.

Statements vs. Expressions

  • Statements: Instructions that perform an action and do not return a value.
  • Expressions: Evaluate to a resulting value. They do not end in semicolons.
Return Values: Functions return the value of the final expression in their block. You can return early using the return keyword. 
fn plus_one(x: i32) -> i32 {
    x + 1 // Note: No semicolon makes this an expression!
}

3.4 Comments

Rust uses // for line comments. For documentation, Rust uses /// (which supports Markdown!), but for general logic explanation, standard line comments are the convention.

3.5 Control Flow

Rust provides if expressions and several types of loops to control execution.

if Expressions

The condition must be a bool. Rust will not automatically try to convert non-Boolean types to a Boolean (unlike JS or C).

Loops

  • loop: Executes forever until you explicitly break.
  • while: Runs while a condition remains true.
  • for: The most common loop. Ideal for iterating through collections.
let a = [10, 20, 30];
for element in a {
    println!("Value: {element}");
}
