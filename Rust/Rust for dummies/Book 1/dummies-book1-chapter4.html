Controlling the Flow

Book 1, Chapter 4 of Rust All-in-One For Dummies covers controlling execution flow: branches using if, else if, and else expressions, using if in let statements, as well as looping structures including loop, while, and for.

1. Basic Conditional Branching

An if expression checks a condition. Unlike some other languages, the condition in Rust must evaluate strictly to a bool—Rust does not automatically convert non-booleans to booleans.

src/main.rs — Branching
Type the if statement condition header checking if number < 10:
if number < 10 {

2. Multiple Conditions with else if

You can combine multiple conditions using else if. Rust executes the block corresponding to the first condition that evaluates to true.

src/main.rs — Multiple Conditions
Type an else if check testing if number % 3 == 0:
else if number % 3 == 0 {

3. Using if in a let Statement

Because if is an expression in Rust, you can use it on the right side of a let statement to assign values dynamically. Note: Both arms must return the same data type.

src/main.rs — Ternary-style Assign
Type a statement binding number to 5 if condition is true, else 6:
let number = if condition { 5 } else { 6 };

4. Repeating Code with loop

The loop keyword tells Rust to execute a block of code over and over again forever until you explicitly instruct it to stop using break.

src/main.rs — Standard Loop
Type a break statement that breaks out of a loop passing the value counter * 2:
break counter * 2;

5. Returning Values from Loops

One of the uses of a loop is to retry an operation. You can pass the result of that operation out of the loop by adding the expression after the break keyword.

src/main.rs — Loop Assignment

let result = loop { break 20; };

6. Conditional Loops with while

A program often needs to evaluate a condition within a loop. While the condition is true, the loop runs; when the condition turns false, the loop ends.

7. Iterating Through Collections with for

Instead of using a index counter with a while loop (which can cause runtime panics if bounds are wrong), the for loop provides safety and efficiency when iterating over arrays or collections.

8. Counting with Range Expressions

Rust provides ranges like 1..4 (generates 1, 2, 3) or (1..4).rev() to reverse the order. This is the safest way to execute code a specific number of times.

src/main.rs — Range Countdown
Type a for loop header counting down using (1..4).rev():
