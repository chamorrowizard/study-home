ES6 Modules Cheat Sheet

Based on The Odin Project curriculum.

Fundamentals

What are ES6 Modules?

ES6 Modules (ESM) allow you to split your JavaScript into multiple files. Unlike the global scope problem of the past, ESM provides each file with its own private scope.

  • Automatic Defer: type="module" scripts are deferred by default.
  • No Global Pollution: Variables are not accessible in the global scope.

Export & Import Patterns

Named Exports

Use for multiple exports per file. Names must match during import.

// math.js
export const add = (a, b) => a + b;

// main.js
import { add } from './math.js';

Default Exports

Use for a single primary export. You can name the import anything.

// User.js
export default class User {...}

// main.js
import Person from './User.js';

Comparison: ESM vs. CommonJS

While ESM is for the browser, CommonJS (CJS) is the standard for Node.js.

Feature ES6 Modules (ESM) CommonJS (CJS)
Keywords import / export require / module.exports
Environment Browsers & Modern Node Node.js Legacy/Standard
Loading Asynchronous Synchronous

Implementation

Linking in HTML

You only need to link the entry point. The browser handles the rest.

<script src="main.js" type="module"></script>

Note: Requires a local server (like Live Preview) to run due to security restrictions.

Advanced Concepts

1. Beyond Basic Imports

While standard imports are common, real-world projects often require more flexible syntax for handling naming collisions and large libraries.

Namespace Imports

Import every export from a file as a single object. Great for utility libraries.

// Importing everything as 'MathObj'
import * as MathObj from './math.js';

console.log(MathObj.add(1, 2));
console.log(MathObj.subtract(5, 2));

Aliases (Renaming)

Use as to rename imports and avoid conflicts with local variables.

// Renaming 'speak' to 'talk'
import { speak as talk } from './user.js';

const speak = "I am a local string";
talk(); // Calls the imported function

2. Technical Mechanics

Execution Timing

Modules are singleton. If you import the same module into 5 different files, the code inside that module only executes once.

Pro Tip: Use modules for configuration files or API clients where you want a single shared instance across your app.

Static vs Dynamic

Standard import statements are static. They must be at the top level and cannot be inside if statements or loops.

// ‚ùå This will throw an error
if (userLoggedIn) {
   import { dashboard } from './ui.js';
}

4. Critical Implementation Rules

Security & Browsers

The CORS Hurdle

Because modules are loaded as separate files via HTTP requests, browsers enforce CORS security. This means you cannot test modules by double-clicking an index.html file.

You must use a local development server such as:

  • [Live Preview](https://marketplace.visualstudio.com/items?itemName=ms-vscode.live-server) for VS Code
  • Python's http.server
  • Node's live-server
