💻 Chapter Overview: Security Control Classification

Security controls are safeguards or countermeasures prescribed for an information system to protect the confidentiality, integrity, and availability of its system and its information. CompTIA Security+ categorizes controls in two main ways: by Category (how the control is implemented) and by Functional Type (what goal the control achieves).

1. Control Categories

Implementation Method

Controls are categorized by how they are implemented: Technical (hardware/software like firewalls), Managerial (administrative oversight and policies), Operational (human-executed procedures like security guards or training), and Physical (real-world items like locks and fences).

~/security/controls/categories.sh
Category: Technical | Managerial | Operational | Physical
>

2. Preventive & Deterrent Controls

Functional Type

Preventive controls stop an incident from occurring in the first place (e.g., IPS, firewalls, mandatory access controls). Deterrent controls discourage potential attackers from attempting a breach by warning them of consequences (e.g., warning signs, visible cameras).

~/security/controls/preventive_deterrent.sh
Type: Preventive blocks attacks | Deterrent discourages attempts
>

3. Detective & Corrective Controls

Functional Type

Detective controls identify and trigger alerts during or after an incident occurs (e.g., IDS, audit logs, motion detectors). Corrective controls mitigate damage, restore operations, and fix systems after an incident has taken place (e.g., backups, patch installation, system images).

~/security/controls/detective_corrective.sh
Type: Detective identifies activity | Corrective restores systems
>

4. Compensating & Directive Controls

Functional Type

Compensating controls provide an alternative solution when a primary control is unfeasible or too costly (e.g., using a sandbox environment when a legacy system cannot be patched). Directive controls direct human behavior to ensure compliance with security requirements (e.g., Acceptable Use Policies, mandatory training).

~/security/controls/compensating_directive.sh
Type: Compensating alternate option | Directive enforces policy
>

5. Managerial & Administrative Controls

Control Category

Managerial controls focus on security governance, risk management, and organizational oversight. They define the rules, policies, frameworks, and assessment strategies used by management (e.g., conducting risk assessments, policy creation, security audits).

~/security/controls/managerial.sh
Managerial: Governance | Risk Assessments | Audits | Policies
>

6. Operational Controls

Control Category

Operational controls are implemented and executed by people rather than automated technical systems. Examples include user security awareness training, disaster recovery plan execution, visitor logging, and security guard patrols.

~/security/controls/operational.sh
Operational: Security Training | Guard Patrols | Backup Execution
>

7. Physical Security Controls

Control Category

Physical controls impact the real-world physical environment to protect hardware, facility perimeters, and personnel. Examples include vehicle bollards, fencing, biometric door locks, HVAC systems, and CCTV cameras.

~/security/controls/physical.sh
Physical: Bollards | Fencing | Biometric Locks | CCTV
>

8. Multi-Attribute Scenario Assessment

Exam Scenario Practice

Every real-world safeguard carries both a Category (how it is implemented) and a Functional Type (its objective). For example, an Intrusion Prevention System (IPS) is Technical + Preventive, while a security guard warning sign is Physical + Deterrent.

IPS = Technical Preventive | Guard Sign = Physical Deterrent
